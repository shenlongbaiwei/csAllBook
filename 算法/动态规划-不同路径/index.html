    动态规划-不同路径¶

    力扣对应题目的链接：

    https://leetcode.cn/problems/unique-paths/description/

    解的代码：

    Text Only
    int uniquePaths(int m, int n) {
        vector<vector<int>> dp(m,vector<int>(n,0));
        for(int i=0;i<m;i++){
            dp[i][0]=1;
        }
        for(int j=0;j<n;j++){
            dp[0][j]=1;
        }
        for(int i=1;i<m;i++){
            for(int j=1;j<n;j++){
                dp[i][j]=dp[i][j-1]+dp[i-1][j];
            }
        }
        return dp[m-1][n-1];
    }

    题目示例：

    这道题目要求求解机器人从左上角到达右下角五角星位置的所有走的路径总数。

    遇到这种在网格中移动求解总数的问题，一般有深度搜索，动态规划，数论这几种方法。

    在这里我着重复盘我个人做这个题的动态规划思路。

    思路¶

    首先，机器人开始能选择的是从右或者从下走。

    重要的是，要能够看出来机器人在第一行，和第一列走的过程中，每一个格子的路径方法都只依赖于左边的路，或者上面的路，都只依赖于一条路径，而其他情况下，则需要依赖格子左边和上边的路径总和的影响，因此这个题的情况需要考虑三种。

    1、第一行，机器人一直向右走，第一行每个元素的路径数都为1.

    2、第一列，机器人一直向下走，第一列的每个元素的路径数都为1.

    3、其余位置，每个格子的dp[i][j]=dp[i-1][j]+dp[i][j-1];

    这个地方的dp数组表示网格中每个位置的路径总和，dp[i][j]表示机器人到达i,j坐标可以选择所有可能的路径数总和。

    因此，我们在实现的时候需要考虑三种情况，并将前两种情况进行单独的for循环就可以实现初始化。

    其余位置进行两层for循环，嵌套运行即可。

    题目来源：

    https://leetcode.cn/problems/unique-paths-ii/

    题目答案代码：

    C++
    int uniquePathsWithObstacles(vector<vector<int>>& obstacleGrid) {
        int m=obstacleGrid.size();
        int n=obstacleGrid[0].size();
        if (obstacleGrid[m-1][n-1] == 1 || obstacleGrid[0][0] == 1) //如果在起点或终点出现了障碍，直接返回0
            return 0;
        vector<vector<int>> dp(m,vector<int>(n,0));

        for(int i=0;i<m&&obstacleGrid[i][0]==0;i++) dp[i][0]=1;
        for(int j=0;j<n&&obstacleGrid[0][j]==0;j++) dp[0][j]=1;
        for(int i=1;i<m;i++){
            for(int j=1;j<n;j++){
                if (obstacleGrid[i][j] == 1) continue;
                    dp[i][j]=dp[i-1][j]+dp[i][j-1];

            }
        }
        return dp[m-1][n-1];
    }
